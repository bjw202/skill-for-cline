# 보고서 → 슬라이드 기획 스킬

보고용 raw text를 분석하여 체계적인 프레젠테이션 기획안(markdown)을 생성한다.
실제 PPT 코드 생성은 `ppt-design-system` 스킬이 담당하며, 이 스킬은 **콘텐츠 전략과 슬라이드 설계**에 집중한다.

---

## 워크플로우 (4단계)

### 1단계: 원문 분석 (Analyze)

사용자가 제공한 raw text를 다음 관점에서 분석한다:

**구조 분석:**
- 핵심 주제(Main Topic)와 하위 주제(Sub-topics) 식별
- 논리적 흐름 파악 (시간순, 인과관계, 비교, 문제-해결)
- 섹션/단락 구분 및 계층 구조 매핑

**콘텐츠 유형 분류:**
- 정량 데이터: 숫자, 비율, 금액, 증감률 → 차트/KPI 후보
- 비교 데이터: A vs B, 전후 비교, 경쟁 분석 → 테이블/차트 후보
- 프로세스: 단계, 절차, 타임라인 → 타임라인/플로우 후보
- 핵심 메시지: 결론, 제안, 액션 아이템 → 강조 슬라이드 후보
- 텍스트 서술: 배경, 설명, 근거 → 본문/카드 레이아웃 후보

**데이터 포인트 추출:**
- 핵심 지표(KPI) 식별 및 목록화
- 비교 가능한 수치 쌍 식별
- 추이/변화 데이터 식별

분석 완료 후 사용자에게 **분석 요약 리포트**를 먼저 제시한다:

```markdown
## 원문 분석 결과

### 식별된 핵심 주제
1. [주제 1]
2. [주제 2]
...

### 콘텐츠 유형 분포
- 정량 데이터: N건 (차트/KPI 후보)
- 비교 데이터: N건 (테이블 후보)
- 프로세스: N건 (타임라인 후보)
- 서술 텍스트: N건 (본문 레이아웃 후보)

### 추출된 핵심 지표
| 지표명 | 값 | 맥락 |
|--------|-----|------|
| [지표] | [값] | [설명] |
```

---

### 2단계: 맥락 질문 (Clarify)

원문 분석 후 `ask_followup_question`으로 다음 항목을 확인한다. 한 번에 최대 3개씩 질문한다.

**필수 질문 (반드시 확인):**

Q1. 발표 목적과 대상
```
이 발표의 목적과 대상 청중은 누구인가요?
예시:
- 경영진 보고 (임원/C-level)
- 팀 내부 공유 (동료/팀원)
- 고객/외부 제안 (클라이언트)
- 교육/워크숍 (학습자)
- 투자자 피칭 (IR)
```

Q2. 발표 시간 및 슬라이드 규모
```
발표 시간과 원하는 슬라이드 수가 있나요?
예시:
- 5분 내외 (5~8장) — 핵심 요약
- 15분 내외 (10~15장) — 표준 보고
- 30분 내외 (15~25장) — 상세 보고
- 자유 (적절히 구성)
```

Q3. 강조하고 싶은 핵심 메시지
```
이 발표에서 가장 전달하고 싶은 핵심 메시지 1~3가지는 무엇인가요?
(원문에서 추출한 후보를 제시하고 사용자에게 확인/수정 요청)
```

**선택 질문 (필요 시):**
- 톤앤매너: 격식체/캐주얼, 보수적/트렌디
- 브랜드 색상 또는 선호 색상 팔레트
- 특정 슬라이드 유형 요청 (예: 반드시 타임라인 포함)
- 제외할 내용이 있는지 확인
- 첨부 데이터(엑셀, 이미지 등)가 있는지 확인

---

### 3단계: 콘텐츠 구조화 (Structure)

질문 응답을 반영하여 원문을 슬라이드 단위로 재구성한다.

**구조화 원칙:**

1. **한 슬라이드 = 한 메시지**: 하나의 슬라이드에 하나의 핵심 메시지만 담는다
2. **피라미드 구조**: 결론 → 근거 → 세부사항 순서로 배치
3. **6x6 규칙**: 슬라이드당 최대 6개 글머리, 글머리당 한글 18자 이내
4. **데이터 우선**: 수치가 있으면 시각화를 우선 고려

**슬라이드 유형 배정 기준:**

| 콘텐츠 유형 | 권장 슬라이드 유형 | 디자인 패턴 |
|-------------|-------------------|------------|
| 도입/표지 | Title Slide | 전체 다크 배경, 중앙 제목 |
| 섹션 전환 | Section Divider | 좌우 분할, 섹션 번호 |
| 핵심 KPI | KPI Dashboard | 숫자 카드 3~4개 + 차트 |
| 데이터 비교 | Data Table | 네이티브 테이블, 8행 이내 |
| 수치 추이 | Chart + Insight | 좌 60% 차트 + 우 40% 인사이트 |
| 프로세스/일정 | Timeline | 세로 타임라인 + 단계별 설명 |
| 다중 항목 | Card Grid | 2x2 또는 2x3 카드 |
| 비교 분석 | Two Column | 좌우 대비 레이아웃 |
| 결론/제안 | Key Takeaway | 핵심 메시지 강조 + 액션 아이템 |
| 상세 설명 | Body Text | 제목 바 + 본문 단락 |

**슬라이드 순서 템플릿:**

경영진 보고:
```
표지 → 핵심 요약(KPI) → 상세 분석(2~3장) → 이슈/리스크 → 액션 플랜 → 마무리
```

프로젝트 현황:
```
표지 → 전체 요약 → [섹션 구분 → 상세 내용] × N → 일정 → 이슈 → 다음 단계
```

제안/피칭:
```
표지 → 문제 정의 → 해결책 → 근거/데이터 → 실행 계획 → 기대 효과 → CTA
```

---

### 4단계: 기획안 출력 (Design Plan)

최종 기획안을 markdown으로 출력한다. 각 슬라이드별로 다음 요소를 포함한다:

**기획안 출력 형식:**

```markdown
# [발표 제목] — 슬라이드 기획안

## 기획 개요
- 대상: [청중]
- 목적: [발표 목적]
- 슬라이드 수: [N]장
- 예상 발표 시간: [N]분
- 색상 팔레트: [선택된 팔레트]

---

## 슬라이드 1: 표지
- **유형**: Title Slide
- **레이아웃**: 전체 다크 배경 (#1A1F36)
- **콘텐츠**:
  - 메인 제목: "[제목]" (44pt, ExtraBold, 흰색)
  - 부제목: "[부제]" (20pt, Regular, 70% 흰색)
  - 하단: "[발표자] | [날짜]" (14pt, 50% 흰색)
- **디자인 노트**: accent 라인(시안) 제목 하단에 배치

---

## 슬라이드 2: 핵심 요약
- **유형**: KPI Dashboard
- **레이아웃**: 제목 바 + KPI 카드 3개 + 하단 차트
- **콘텐츠**:
  - KPI 1: [지표명] — [값] (48pt, Blue)
  - KPI 2: [지표명] — [값] (48pt, Cyan)
  - KPI 3: [지표명] — [값] (48pt, Yellow)
  - 차트: [차트 유형] — [데이터 설명]
- **디자인 노트**: KPI 카드는 라운드 모서리, 각각 다른 accent 색상

---
...이하 반복...
```

**디자인 요소 표기법:**

각 슬라이드에 다음 디자인 정보를 명시한다:
- **레이아웃 유형**: 8개 패턴 중 선택 (표지/섹션/테이블/차트/카드/타임라인/KPI/2단)
- **색상**: 배경색, 텍스트색, 강조색을 hex 코드로 표기
- **타이포그래피**: 폰트 크기, 굵기, 색상
- **시각화 유형**: 차트 종류(BAR/LINE/PIE/DOUGHNUT), 테이블 구조
- **배치**: 요소의 대략적 위치 (상단/하단/좌측/우측/중앙)
- **여백/간격**: 요소 간 간격 지침

---

## 콘텐츠 변환 규칙

### 원문 → 슬라이드 변환 패턴

**긴 문단 → 글머리 기호:**
- 원문에서 핵심 키워드와 수치를 추출
- 한글 18자 이내로 압축
- 최대 6개 항목으로 제한

**나열형 데이터 → 테이블:**
- 3개 이상 항목이 동일 속성으로 비교 가능하면 테이블
- 열 수 최대 5~6개, 행 수 최대 8개
- 초과 시 슬라이드 분할 명시

**수치 데이터 → 차트 유형 결정:**

| 데이터 특성 | 권장 차트 | 이유 |
|------------|----------|------|
| 시간별 변화 | LINE | 추이 파악에 최적 |
| 항목별 크기 비교 | BAR | 크기 차이 직관적 |
| 구성 비율 | PIE / DOUGHNUT | 전체 대비 비중 |
| 다차원 비교 | RADAR | 여러 축 동시 비교 |
| 분포/상관 | SCATTER | 데이터 패턴 탐색 |

**핵심 수치 → KPI 카드:**
- 전체 맥락에서 가장 중요한 수치 3~4개 선별
- 단위 포함 (원, %, 건, 명 등)
- 변화율이 있으면 함께 표기 (전년비 +18%)

---

## 청중별 톤 가이드

| 청중 | 슬라이드 밀도 | 데이터 비중 | 톤 |
|------|-------------|-----------|-----|
| 경영진 (C-level) | 낮음 (5~10장) | 높음 (KPI 중심) | 결론 우선, 간결 |
| 실무진 (팀원) | 중간 (10~20장) | 중간 | 상세 근거 포함 |
| 고객/외부 | 중간 (10~15장) | 중간 | 전문적이나 이해하기 쉽게 |
| 투자자 (IR) | 낮음 (8~12장) | 높음 | 성장성/수익성 강조 |
| 교육/워크숍 | 높음 (15~30장) | 낮음 | 단계적, 예시 풍부 |

---

## 슬라이드 수 산정 기준

원문 분량과 발표 시간에 따른 권장 슬라이드 수:

| 발표 시간 | 슬라이드 수 | 슬라이드당 시간 |
|----------|-----------|---------------|
| 5분 | 5~8장 | 40~60초 |
| 10분 | 8~12장 | 50~75초 |
| 15분 | 12~18장 | 50~75초 |
| 30분 | 18~25장 | 70~100초 |
| 60분 | 25~40장 | 90~140초 |

**자동 산정 공식:**
- 기본: `발표시간(분) × 0.8 ~ 1.2` 장
- 데이터 밀도 높으면: 장당 시간 늘림 (데이터 슬라이드 = 1.5분)
- 섹션 구분 슬라이드 추가: `주요 섹션 수 - 1`장

## ppt-design-system 스킬 연계

이 스킬의 출력물(markdown 기획안)을 `ppt-design-system` 스킬에 전달하면
실제 PptxGenJS 코드를 생성할 수 있다.

**연계 흐름:**
1. 이 스킬: raw text → 분석 → 질문 → 구조화 → **markdown 기획안**
2. ppt-design-system: markdown 기획안 → **PptxGenJS 코드** → .pptx 파일

기획안의 레이아웃 유형명은 `ppt-design-system`의 8개 패턴명과 일치시킨다:
- Title Slide, Section Divider, Two Column, Card Grid
- Timeline, KPI Dashboard, Data Table, Chart + Insight
