# 슬라이드 구성 템플릿 모음

SKILL.md의 3단계(Structure)에서 참조한다.

---

## 청중별 템플릿

### T1: 경영진 보고 (Executive Report)

**특성:** 결론 우선, 간결, KPI 중심, 5~10장

```
슬라이드 구성:
1. [Title Slide]     표지
2. [KPI Dashboard]   핵심 요약 (3~4개 KPI + 요약 차트)
3. [Chart+Insight]   주요 실적 상세 (차트 + 핵심 인사이트)
4. [Data Table]      비교 분석 (필요 시, 핵심 항목만)
5. [Card Grid]       이슈 & 리스크 (2~4개 카드)
6. [Timeline]        향후 계획 / 액션 플랜
7. [Key Takeaway]    마무리: 핵심 메시지 + 요청 사항
```

**디자인 가이드:**
- 색상: 차분한 네이비 기반 (bg_dark + accent_blue)
- 폰트: 큰 KPI 숫자(48pt+), 본문 최소화
- 여백: 넓게, 한 슬라이드에 과적 금지
- 차트: 단순한 BAR/LINE, 범례 최소화

---

### T2: 팀 내부 공유 (Team Sharing)

**특성:** 상세 근거 포함, 실무 수준, 10~20장

```
슬라이드 구성:
1.  [Title Slide]       표지
2.  [Body Text]         배경 및 목적
3.  [Section Divider]   섹션 1 구분
4.  [KPI Dashboard]     현황 데이터
5.  [Chart+Insight]     상세 분석 1
6.  [Data Table]        상세 데이터
7.  [Section Divider]   섹션 2 구분
8.  [Two Column]        비교 분석
9.  [Card Grid]         주요 발견사항
10. [Section Divider]   섹션 3 구분
11. [Timeline]          실행 계획
12. [Data Table]        상세 일정/담당
13. [Card Grid]         리스크 & 대응
14. [Key Takeaway]      결론 및 논의 사항
```

**디자인 가이드:**
- 색상: 밝은 톤 (bg_primary + 다양한 accent)
- 테이블 활용 빈도 높음
- 상세 데이터 제시 가능 (부록 포함)
- 논의 포인트 명시

### T3: 고객/외부 제안 (Client Proposal)

**특성:** 전문적이면서 이해하기 쉽게, 10~15장

```
슬라이드 구성:
1.  [Title Slide]       표지 (회사 로고 포함)
2.  [Body Text]         문제 정의 (고객의 Pain Point)
3.  [KPI Dashboard]     현황 데이터 / 시장 상황
4.  [Section Divider]   솔루션 소개
5.  [Card Grid]         핵심 제안 (3~4개 카드)
6.  [Two Column]        기존 방식 vs 제안 방식
7.  [Chart+Insight]     기대 효과 (ROI/성과 예측)
8.  [Timeline]          실행 로드맵
9.  [Data Table]        비용/투자 계획 (필요 시)
10. [Card Grid]         레퍼런스 / 성공 사례
11. [Key Takeaway]      마무리: CTA + 연락처
```

**디자인 가이드:**
- 색상: 고객 브랜드 색상 반영 또는 Warm Corporate 팔레트
- 이미지 활용 적극적 (제품/서비스 스크린샷)
- 전문 용어 최소화, 도식화 우선
- CTA 명확하게

### T4: 투자자 피칭 (Investor Pitch)

**특성:** 성장성/수익성 강조, 8~12장, 스토리 중심

```
슬라이드 구성:
1.  [Title Slide]       표지 (로고 + 한줄 설명)
2.  [Body Text]         문제 정의 (시장 기회)
3.  [KPI Dashboard]     시장 규모 (TAM/SAM/SOM)
4.  [Card Grid]         솔루션 + 차별점 (3~4개)
5.  [Chart+Insight]     트랙션 / 성장 지표
6.  [Two Column]        비즈니스 모델 (수익 구조)
7.  [Timeline]          로드맵 / 마일스톤
8.  [KPI Dashboard]     재무 전망 (매출/비용/이익)
9.  [Card Grid]         팀 소개 (핵심 멤버)
10. [Key Takeaway]      투자 요청 (금액/용도/기대효과)
```

**디자인 가이드:**
- 색상: 신뢰감 있는 블루 계열 또는 트렌디한 팔레트
- KPI 숫자 크게 강조 (성장률, 매출 등)
- 차트: 우상향 트렌드 강조
- 마지막 슬라이드: 명확한 Ask (투자금액/조건)

### T5: 교육/워크숍 (Training/Workshop)

**특성:** 단계적, 예시 풍부, 15~30장

```
슬라이드 구성:
1.  [Title Slide]       표지
2.  [Body Text]         학습 목표 (3~4개)
3.  [Timeline]          전체 일정 / 아젠다
4.  [Section Divider]   모듈 1
5.  [Body Text]         개념 설명
6.  [Two Column]        예시: Before vs After
7.  [Card Grid]         핵심 포인트 정리
8.  [Section Divider]   모듈 2
9-12. ...반복...
13. [Card Grid]         핵심 요약 (전체 모듈)
14. [Body Text]         실습 과제 / 다음 단계
15. [Key Takeaway]      Q&A / 연락처
```

**디자인 가이드:**
- 색상: 친근한 톤 (밝은 배경 + 다양한 accent)
- 텍스트 분량 적절히 (설명이 많으므로)
- 섹션 구분 명확 (모듈별 색상 차별화 가능)
- 실습/활동 슬라이드는 배경색 변경으로 구분

## 목적별 템플릿

### P1: 월간/분기 정기 보고

```
1. 표지 (기간 명시)
2. 핵심 KPI 요약 (전기 대비)
3. [섹션] 매출/실적 → 차트 + 테이블
4. [섹션] 운영/프로세스 → KPI + 이슈
5. [섹션] 인력/조직 → 카드 그리드
6. 이슈 & 리스크
7. 다음 기간 계획
8. 마무리
```

### P2: 프로젝트 킥오프

```
1. 표지 (프로젝트명)
2. 프로젝트 배경 & 목표
3. 범위 (In-scope / Out-of-scope) → Two Column
4. 팀 구성 → Card Grid
5. 마일스톤 → Timeline
6. 리스크 & 대응 → Data Table
7. 커뮤니케이션 계획
8. Q&A
```

### P3: 장애/사고 보고 (Incident Report)

```
1. 표지 (긴급 표시)
2. 사고 개요 (When/What/Impact) → KPI Dashboard
3. 타임라인 (발생~복구) → Timeline
4. 근본 원인 분석 → Two Column (증상 vs 원인)
5. 대응 조치 → Card Grid
6. 재발 방지 대책 → Data Table
7. 후속 조치 일정 → Timeline
```

### P4: 시장/경쟁 분석

```
1. 표지
2. 시장 규모 & 성장률 → Chart + Insight
3. 주요 트렌드 → Card Grid (3~4개)
4. 경쟁사 비교 → Data Table + RADAR Chart
5. SWOT 분석 → Card Grid (2x2)
6. 기회 & 위협 → Two Column
7. 전략적 시사점 → Key Takeaway
```

### P5: 성과/실적 리뷰

```
1. 표지 (기간)
2. 핵심 성과 요약 → KPI Dashboard (목표 vs 실적)
3. 상세 실적 → Chart + Insight (추이 그래프)
4. 목표 달성 현황 → Data Table (항목별 달성률)
5. 우수 사례 → Card Grid
6. 개선 필요 영역 → Two Column (현재 vs 목표)
7. 다음 기간 목표 → KPI Dashboard
8. 마무리
```

## 공통 슬라이드 패턴

### 강조 슬라이드 (Key Takeaway)

```markdown
- **레이아웃**: 중앙 정렬, 넓은 여백
- **배경**: bg_secondary (#F5F7FA) 또는 accent 색상 배경
- **메인 텍스트**: 28~36pt, ExtraBold, text_primary
- **서브 텍스트**: 16~18pt, Regular, text_secondary
- **accent**: 하단에 작은 accent 바 또는 아이콘
```

### 전환 질문 슬라이드 (Transition Question)

```markdown
- **레이아웃**: 중앙 정렬
- **배경**: bg_dark (#1A1F36)
- **텍스트**: 인용구 스타일 (ChosunilboNM, 22pt, 흰색)
- **예시**: "그렇다면, 우리의 전략은 무엇인가?"
```

### CTA 슬라이드 (Call to Action)

```markdown
- **레이아웃**: 중앙 정렬 + 하단 액션
- **메인**: 핵심 요청/제안 (28pt, Bold)
- **액션 아이템**: 3~4개 글머리 기호 (구체적 행동)
- **연락처/다음 미팅**: 하단에 배치
```

## 슬라이드 전환 연결어

| 전환 유형 | 연결 문구 예시 |
|----------|--------------|
| 현황 → 분석 | "이러한 현황의 원인을 살펴보면..." |
| 데이터 → 인사이트 | "이 데이터가 말해주는 것은..." |
| 문제 → 해결 | "이 문제를 해결하기 위해..." |
| 과거 → 미래 | "지금까지의 성과를 바탕으로..." |
| 분석 → 제안 | "따라서 다음과 같이 제안합니다" |
| 일반 → 구체 | "구체적인 실행 방안은..." |

## 콘텐츠 변환 규칙

### 원문 → 슬라이드 변환 패턴

**긴 문단 → 글머리 기호:**
- 원문에서 핵심 키워드와 수치를 추출
- 한글 18자 이내로 압축
- 최대 6개 항목으로 제한

**나열형 데이터 → 테이블:**
- 3개 이상 항목이 동일 속성으로 비교 가능하면 테이블
- 열 수 최대 5~6개, 행 수 최대 8개
- 초과 시 슬라이드 분할 명시

**수치 데이터 → 차트 유형 결정:**

| 데이터 특성 | 권장 차트 | 이유 |
|------------|----------|------|
| 시간별 변화 | LINE | 추이 파악에 최적 |
| 항목별 크기 비교 | BAR | 크기 차이 직관적 |
| 구성 비율 | PIE / DOUGHNUT | 전체 대비 비중 |
| 다차원 비교 | RADAR | 여러 축 동시 비교 |
| 분포/상관 | SCATTER | 데이터 패턴 탐색 |

**핵심 수치 → KPI 카드:**
- 전체 맥락에서 가장 중요한 수치 3~4개 선별
- 단위 포함 (원, %, 건, 명 등)
- 변화율이 있으면 함께 표기 (전년비 +18%)

## 청중별 톤 가이드

| 청중 | 슬라이드 밀도 | 데이터 비중 | 톤 |
|------|-------------|-----------|-----|
| 경영진 (C-level) | 낮음 (5~10장) | 높음 (KPI 중심) | 결론 우선, 간결 |
| 실무진 (팀원) | 중간 (10~20장) | 중간 | 상세 근거 포함 |
| 고객/외부 | 중간 (10~15장) | 중간 | 전문적이나 이해하기 쉽게 |
| 투자자 (IR) | 낮음 (8~12장) | 높음 | 성장성/수익성 강조 |
| 교육/워크숍 | 높음 (15~30장) | 낮음 | 단계적, 예시 풍부 |

## 슬라이드 수 산정 기준

| 발표 시간 | 슬라이드 수 | 슬라이드당 시간 |
|----------|-----------|---------------|
| 5분 | 5~8장 | 40~60초 |
| 10분 | 8~12장 | 50~75초 |
| 15분 | 12~18장 | 50~75초 |
| 30분 | 18~25장 | 70~100초 |
| 60분 | 25~40장 | 90~140초 |

**자동 산정 공식:**
- 기본: `발표시간(분) x 0.8 ~ 1.2`장
- 데이터 밀도 높으면: 장당 시간 늘림 (데이터 슬라이드 = 1.5분)
- 섹션 구분 슬라이드 추가: `주요 섹션 수 - 1`장
