# 원문 분석 체크리스트

SKILL.md의 1단계(Analyze)를 수행할 때 참조하는 상세 체크리스트.

---

## 1차 스캔: 문서 메타 정보

원문을 처음 읽을 때 파악할 항목:

| 항목 | 확인 내용 | 메모 |
|------|----------|------|
| 문서 유형 | 보고서/기획안/회의록/분석서/제안서 | |
| 분량 | 총 글자수, 페이지 수 추정 | |
| 작성 시점 | 날짜, 기간, 분기 | |
| 작성 목적 | 보고/제안/공유/교육/승인 요청 | |
| 대상 독자 | 경영진/실무진/외부/전체 | |

---

## 2차 스캔: 콘텐츠 유형 분류

원문의 각 단락/섹션을 아래 6가지 유형으로 분류한다:

### 유형 A: 정량 데이터 (Quantitative)

**식별 기준:**
- 숫자, 금액, 비율, 백분율 포함
- 단위가 명시됨 (원, 건, 명, %, 개, 시간 등)
- "증가/감소/달성/초과/미달" 등 변화 표현

**추출 항목:**
- 지표명 (예: 매출, 이용자 수, 전환율)
- 수치 값 (예: 3.2억, 15%, 1,234건)
- 비교 기준 (예: 전년 동기, 목표 대비, 전월 대비)
- 변화 방향 (증가/감소/유지)

**슬라이드 매핑:**
- 핵심 지표 3~4개 → KPI Dashboard
- 시계열 데이터 → LINE/AREA Chart + Insight
- 항목별 비교 → BAR Chart
- 구성비 → PIE/DOUGHNUT Chart

### 유형 B: 비교 데이터 (Comparative)

**식별 기준:**
- A vs B 구조
- "대비/비교/차이/우위/열위" 표현
- 2개 이상 항목이 동일 기준으로 나열

**추출 항목:**
- 비교 대상 (예: 자사 vs 경쟁사, 안A vs 안B)
- 비교 기준 (예: 가격, 품질, 기능, 성능)
- 각 대상별 값

**슬라이드 매핑:**
- 2~4개 항목 비교 → Two Column 또는 Data Table
- 다차원 비교 → RADAR Chart
- 장단점 비교 → Card Grid (장점/단점 카드)

### 유형 C: 프로세스/일정 (Process)

**식별 기준:**
- 순서, 단계, 절차 표현
- "1단계/2단계", "먼저/다음/마지막"
- 날짜, 기간, 마일스톤 포함
- "완료/진행중/예정" 등 상태 표현

**추출 항목:**
- 단계명 또는 마일스톤명
- 기간 (시작일~종료일)
- 상태 (완료/진행/예정)
- 담당자 (있는 경우)

**슬라이드 매핑:**
- 3~5단계 프로세스 → Timeline
- 일정표 → Data Table (간트 대용)
- 마일스톤 → Timeline + 상태 아이콘

### 유형 D: 핵심 메시지 (Key Message)

**식별 기준:**
- 결론, 제안, 권고, 액션 아이템
- "따라서/결론적으로/제안합니다/요청 사항" 표현
- 문서 서두 또는 말미에 위치하는 요약문

**추출 항목:**
- 핵심 결론 (1~3문장)
- 제안 사항 (구체적 액션)
- 기대 효과 (있는 경우)
- 요청 사항 (승인/예산/인력 등)

**슬라이드 매핑:**
- 핵심 결론 → 강조 슬라이드 (큰 텍스트 + accent 색상)
- 액션 아이템 → 글머리 기호 목록 (체크박스 스타일)
- 요청 사항 → CTA(Call to Action) 슬라이드

### 유형 E: 서술 텍스트 (Narrative)

**식별 기준:**
- 배경 설명, 현황 기술, 근거 제시
- 긴 문단 형태
- 수치보다 설명 중심

**추출 항목:**
- 핵심 키워드 (볼드/강조 처리된 단어)
- 인과관계 ("~때문에/~로 인해/~의 결과")
- 예시나 사례

**슬라이드 매핑:**
- 핵심만 추출 → 글머리 기호 (6x6 규칙)
- 대비 구조 → Two Column
- 여러 소주제 → Card Grid

### 유형 F: 참조/부록 (Reference)

**식별 기준:**
- 출처, 각주, 참고문헌
- 법적 고지, 면책 조항
- 상세 데이터 테이블 (본문보다 세부적)

**추출 항목:**
- 출처 정보
- 상세 데이터 (필요 시 부록 슬라이드)

**슬라이드 매핑:**
- 일반적으로 슬라이드에 포함하지 않음
- 필요 시 부록(Appendix) 섹션으로 분리
- 출처는 각 슬라이드 하단 캡션으로 표기

## 3차 스캔: 논리 구조 파악

### 논리 흐름 유형

| 유형 | 패턴 | 슬라이드 구성 |
|------|------|-------------|
| 시간순 | 과거 → 현재 → 미래 | Timeline 기반 |
| 인과관계 | 원인 → 결과 → 대응 | 문제-해결 구조 |
| 비교 | A vs B → 분석 → 결론 | Two Column 기반 |
| 피라미드 | 결론 → 근거 → 세부 | 결론 우선 배치 |
| MECE | 범주1 + 범주2 + ... | 섹션 분할 구조 |

### 스토리라인 설계

**경영진 보고:**
```
1. 핵심 요약 (So What?) — 결론부터
2. 현황 데이터 (Evidence) — KPI/차트
3. 분석 (Why?) — 원인/배경
4. 제안 (What Next?) — 액션 플랜
5. 마무리 (Ask) — 요청/승인
```

**프로젝트 보고:**
```
1. 프로젝트 개요 — 목표/범위
2. 진행 현황 — KPI/진척률
3. 주요 성과 — 데이터/사례
4. 이슈/리스크 — 문제점/대응
5. 향후 계획 — 일정/마일스톤
6. 요청 사항 — 의사결정/지원
```

**제안서:**
```
1. 문제 정의 — Pain Point
2. 시장/현황 — 데이터
3. 솔루션 — 제안 내용
4. 차별점 — 경쟁 비교
5. 실행 계획 — 로드맵
6. 기대 효과 — ROI/KPI
7. 마무리 — CTA
```

## 4차: 누락 정보 식별

분석 후 기획에 필요하지만 원문에 없는 정보를 식별하여 2단계(Clarify)에서 질문한다:

**자주 누락되는 정보:**
- 비교 기준 시점 (전년? 전월? 전분기?)
- 목표 수치 (달성률 계산 시 필요)
- 데이터 출처/기준일
- 청중 및 발표 맥락
- 우선순위 (어떤 내용이 더 중요한지)
