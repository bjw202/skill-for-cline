# 기술 스택 (Tech Stack)

> 슬기로운 코워크 생활 기술 구성 및 아키텍처 상세

---

## 기술 스택 개요

슬기로운 코워크 생활은 코드 실행 없이 **순수 Markdown 기반 지식 시스템**으로 동작합니다. AI 모델이 내장된 지식, 템플릿, 워크플로우 정의를 해석하여 결과물을 생성하는 방식입니다. 별도 빌드 과정이나 런타임 환경이 필요하지 않으며, 콘텐츠 포맷과 AI 플랫폼 설정만으로 구성됩니다.

---

## 콘텐츠 형식

### Markdown (.md)

프로젝트 콘텐츠의 100%가 Markdown으로 작성됩니다.

- **SKILL.md**: Claude Code 스킬 정의 파일. YAML 프론트매터 + Markdown 본문 구조
- **Knowledge 파일**: Cline이 자동 로드하는 도메인 지식 정의
- **Workflow 파일**: 단계별 실행 절차를 Markdown 목록/헤딩으로 구조화
- **커맨드 파일**: 슬래시 커맨드 동작과 파라미터 정의

Markdown 선택 이유:
- AI 모델이 자연스럽게 파싱하고 해석 가능
- 버전 관리(Git)에 최적화된 텍스트 형식
- 플랫폼 종속성 없음 (Claude Code, Cline 양쪽에서 동일하게 동작)
- 사람이 읽고 수정하기 쉬운 형식

### YAML

설정 파일과 스킬 메타데이터에 사용됩니다.

- **SKILL.md 프론트매터**: 스킬 이름, 설명, 트리거 키워드 정의
- **`.moai/config/sections/*.yaml`**: 프로젝트 레벨 품질 게이트, 언어, 사용자 설정
- **`plugin.json`**: Claude Code 플러그인 마니페스트 (JSON 형식)

## 플랫폼 타겟

### Claude Code (네이티브 플러그인)

**아키텍처**: Progressive Disclosure 3단계 스킬 시스템

```
Level 1 (메타데이터, ~100 토큰)
  - 스킬 이름 및 트리거 키워드
  - 세션 시작 시 항상 로드
  - 메모리 오버헤드 최소화

Level 2 (본문, ~5K 토큰)
  - 상세 지침, 워크플로우, 예시
  - 트리거 감지 시 동적 로드
  - 필요한 시점에만 컨텍스트 소비

Level 3 (번들, 무제한)
  - 참조 문서, 템플릿, 데이터
  - 사용자 요청 또는 Claude 판단 시 로드
  - 대용량 데이터 온디맨드 제공
```

**배포 방식**: Claude 플러그인 마켓플레이스 (`modu-ai/smart-cowork-life`)

**진입점**: `smart-office-life.plugin` — 스킬과 커맨드를 묶은 플러그인 번들

**트리거 메커니즘**: 사용자 입력에서 키워드 감지 → 해당 스킬 자동 활성화 (예: "이메일" 입력 시 비즈니스 이메일 스킬 로드)

### Cline (Rules 변환본)

**아키텍처**: 2계층 Knowledge/Workflow 시스템

```
Tier 1: Knowledge Rules (항시 활성)
  - 10종 *-knowledge.md 파일
  - Cline이 세션 시작 시 자동 로드
  - 도메인 컨텍스트 상시 제공
  - 트리거 없이 모든 대화에 영향

Tier 2: Explicit Workflows (명시적 실행)
  - 13종 workflows/*.md 파일
  - 사용자 명시 요청 시 실행
  - Claude Code 슬래시 커맨드에 대응
  - 단계별 절차 정의 포함
```

**배포 방식**: 프로젝트 루트에 `.clinerules/` 디렉토리 복사

**Claude Code 대비 차이**: 스킬 자동 트리거 기능 없음. Knowledge Rules가 항시 로드되어 있어 관련 요청 시 자동 적용.

## 프레임워크 및 도구

### MoAI-ADK (오케스트레이션 프레임워크)

개발 워크플로우 전반을 관리하는 AI 에이전트 오케스트레이션 프레임워크입니다.

- **역할**: SPEC 작성 → 구현 → 문서화의 3단계 워크플로우 자동화
- **개발 방법론**: TDD (Test-Driven Development) 기본 설정
- **품질 게이트**: TRUST 5 프레임워크 (Tested, Readable, Unified, Secured, Trackable)
- **에이전트 계층**: Manager 8종, Expert 9종, Builder 3종의 전문화된 서브에이전트

설정 파일: `.moai/config/sections/quality.yaml`

### PptxGenJS (PPT 생성 라이브러리)

PPT 디자인 시스템 스킬에서 참조하는 JavaScript 라이브러리입니다.

- **버전**: 최신 안정 버전 (Node.js 환경)
- **용도**: AI가 생성하는 PPT 코드 기반 슬라이드 제작 가이드 제공
- **핵심 API**:
  - `addTable()`: 매출표, 일정표, 비교표 등 구조화 데이터용 네이티브 테이블
  - `addChart()`: BAR/LINE/PIE/DOUGHNUT 등 10종 차트 유형
  - `addImage()`: 이미지·로고·SVG 삽입
  - `addText()`: 글머리 기호, 번호 목록, 인용문
  - `addShape()`: 장식·레이아웃 보조 도형 (장식 전용으로 용도 한정)

스킬 내 역할: 실제 코드 실행이 아닌, AI가 올바른 PptxGenJS 코드를 생성하도록 안내하는 참조 가이드로 활용됩니다.

### Mermaid v11 (다이어그램 도구)

SVG 도식화 스킬에서 사용하는 텍스트 기반 다이어그램 생성 도구입니다.

- **버전**: Mermaid v11 (공식 문법 레퍼런스 내장)
- **지원 다이어그램 12종**:
  - flowchart (플로우차트)
  - sequenceDiagram (시퀀스 다이어그램)
  - classDiagram (클래스 다이어그램)
  - stateDiagram (상태 다이어그램)
  - erDiagram (ER 다이어그램)
  - gantt (간트차트)
  - pie (파이차트)
  - mindmap (마인드맵)
  - timeline (타임라인)
  - gitGraph (Git 그래프)
  - xychart-beta (XY 차트)
  - quadrantChart (사분면 차트)
- **고급 기능**: handDrawn 스타일, ELK 레이아웃 엔진, 조직도·타임라인·도넛차트 SVG 함수

### MCP 서버

개발 과정에서 활용하는 Model Context Protocol 서버입니다.

- **Context7**: 외부 라이브러리 최신 문서를 실시간으로 참조
- **Sequential Thinking**: 복잡한 문제 분석 및 아키텍처 의사결정에 활용

## 데이터 시스템

### Excel 365 동적 배열 함수

엑셀 업무 자동화 스킬에 내장된 최신 Excel 함수 체계입니다.

- **XLOOKUP**: VLOOKUP 대체, 양방향 검색, 에러 처리 내장
- **FILTER**: 조건 기반 동적 필터링
- **SORT / SORTBY**: 다중 기준 정렬
- **UNIQUE**: 중복 제거 고유값 추출
- **SEQUENCE**: 연속 숫자 배열 생성

### 2026년 4대보험 요율

급여 계산 및 엑셀 자동화 스킬에 내장된 최신 요율 데이터입니다.

| 보험 | 근로자 부담 | 사업주 부담 | 비고 |

| 국민연금 | 4.5% | 4.5% | 기준소득월액 기준 |
| 건강보험 | 3.545% | 3.545% | 장기요양보험 별도 |
| 고용보험 | 0.9% | 0.9~3.6% | 사업 규모에 따라 차등 |

### 7개 업종 전문용어 사전

한영 번역 스킬에 내장된 업종별 전문용어 데이터베이스입니다.

5. **물류·유통**: 공급망, 배송, 재고 용어
6. **영업·세일즈**: 계약, 협상, CRM 용어

## 빌드 및 배포

### Claude Code 마켓플레이스 배포

1. `smart-office-life.plugin` 파일 업데이트 (스킬·커맨드 번들)
2. GitHub 저장소 (`modu-ai/smart-cowork-life`) 에 Push
3. 사용자가 Claude 마켓플레이스에서 `modu-ai/smart-cowork-life` 검색 후 설치
4. 설치 즉시 12종 스킬 + 5종 커맨드 활성화

### Cline 배포

1. `.clinerules/` 디렉토리를 ZIP으로 배포
2. 사용자가 프로젝트 루트에 압축 해제
3. Cline이 자동으로 Knowledge Rules 로드 시작

### 콘텐츠 업데이트 주기

- 법정 요율(4대보험 등): 연 1회 (매년 초 업데이트)
- 신규 스킬 추가: 분기별 검토
- 버그 수정 및 개선: 수시

## 성능 고려사항

### 토큰 최적화

Progressive Disclosure 방식으로 컨텍스트 오버헤드를 최소화합니다.

- Level 1 메타데이터: 스킬당 ~100 토큰 (항상 로드)
- Level 2 본문: 스킬당 ~5,000 토큰 (트리거 시 로드)
- Level 3 번들: 무제한 (요청 시 로드)

12종 스킬 전체를 항상 로드할 경우 최대 ~1,200 토큰 소비, 필요한 스킬만 활성화 시 ~5,000~10,000 토큰 수준을 유지합니다.

### Cline 최적화

Knowledge Rules 10종이 항상 로드되므로, 각 파일의 크기를 최소화하여 세션 초기 토큰 사용량을 관리합니다. 복잡한 절차는 Workflow 파일로 분리하여 필요 시에만 로드합니다.
